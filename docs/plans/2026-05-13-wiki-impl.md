# Wiki / Explainer Page Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a static `/wiki` page to the OTB platform sidebar with accordion content covering the OTB process, all field definitions, user roles and permissions, and a glossary.

**Architecture:** Four files — a static content data module (`wiki-content.ts`), two presentational sub-components (`ProcessSteps`, `PermissionMatrix`), and the page itself (`wiki/page.tsx`). The page uses Ant Design `Collapse` for accordion panels and `Anchor` for a sticky right-side table of contents. All content is hardcoded; no API calls.

**Tech Stack:** Next.js 16 App Router, Ant Design 6 (`Collapse`, `Anchor`, `Table`, `Tag`, `Typography`), design tokens from `src/lib/designTokens.ts`

### Task 4: Wiki page

**Files:**
- Create: `src/app/wiki/page.tsx`

The page is a client component (needs scroll-linked Anchor). It composes the four accordion panels and the sticky TOC. The `HEADER_OFFSET` constant (80px) accounts for the 56px top header plus 24px breathing room so anchor targets aren't hidden under the header on scroll.

All `Collapse` items use `forceRender: true` so their inner IDs are always in the DOM (needed for the `Anchor` intersection tracking, even when a panel is collapsed).
